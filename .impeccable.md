# Impeccable Design Context — llm-perf

## Design Context

### Users
- **Primary**: Team leads and post-training algorithm engineers at Huawei
- **Technical level**: High — they understand RL training, parallelism strategies, memory budgets
- **Context**: Planning RL training runs — given constraints (epoch time, data volume, device count, model architecture), find optimal RL training/inference config and TPS targets
- **Job to be done**: Quick, trustworthy analysis that informs resource allocation and configuration decisions

### Brand Personality
**Professional, Insightful, Extensible**

- **Professional**: This is a decision-support tool for leadership — it must look credible and polished, not like a prototype
- **Insightful**: Surface the "so what" — highlight bottlenecks, trade-offs, and recommendations clearly
- **Extensible**: The UI should feel like it can grow (more models, more hardware, more analysis modes) without becoming cluttered

### Emotional Goals
- **Confidence**: Users trust the numbers and act on them
- **Clarity**: Complex multi-dimensional data is immediately understandable
- **Efficiency**: Get to the answer fast, no unnecessary clicks or cognitive load

### Aesthetic Direction

**Visual Tone**: Clean, data-dense professional dashboard — MindSpore Insight meets Grafana

**Theme**: Light mode
- Warm neutral backgrounds (not pure white) — easy on eyes for extended analysis sessions
- Huawei red (`#CF0A2C`) as strategic accent — primary CTAs, key metric highlights, brand touch
- NOT red-dominant; red is used sparingly for emphasis and brand recognition

**Color System**:
- **Neutral base**: Warm off-white backgrounds, tinted gray cards, dark charcoal text
- **Brand accent**: Huawei red `#CF0A2C` for primary buttons, active states, key highlights
- **Data palette** (functional, semantic):
  - Green `#16a34a` → feasible, healthy, within budget
  - Red/Rose `#dc2626` → OOM, infeasible, over budget
  - Amber `#f59e0b` → warnings, budget lines, thresholds
  - Blue `#2563eb` → communication, network ops
  - Purple `#7c3aed` → compute, epoch metrics (kept from current palette)
  - Cyan `#06b6d4` → reference model, secondary data series
- Colors should be slightly desaturated for backgrounds, full saturation only for data points and accents

**Typography**:
- Clean sans-serif with strong hierarchy
- Large bold numbers for KPI cards (the first thing users see)
- Medium-weight section headers
- Compact, readable labels for data-dense areas
- No monospace unless showing actual code/config values

**Layout Patterns**:
- **KPI cards on top** (Grafana-style): Epoch time, Gen TPS, Train TPS, Bottleneck — the headline answer
- **Overview → detail drill-down** (MindSpore Insight-style): Summary first, details on demand
- **Consistent panel grid**: Sections with clear boundaries, collapsible where appropriate
- **Topology as interactive diagram**: Click-to-inspect pattern (Netron-style)

**References** (what to borrow):
- **W&B**: Panel grid layout, consistent color-per-run across charts
- **MindSpore Insight**: Step trace timeline, operator breakdown, tuning suggestions
- **Grafana GPU dashboards**: KPI gauges/cards on top, data-dense but readable
- **LLM-Viewer**: Roofline scatter plot, click-to-drill-down on layers
- **MLflow**: Table-to-chart toggle for comparison views

**Anti-references** (what to avoid):
- Current llm-perf UI (too plain, default Gradio look, no visual hierarchy)
- Generic AI dashboards (cyan-on-dark, purple gradients, glassmorphism)
- Overly decorative UIs that sacrifice data density for aesthetics
- Dark mode "hacker" aesthetics — this is a professional tool for leadership

### Design Principles

1. **Data first**: Every pixel should serve analysis. No decorative elements that don't aid comprehension.
2. **Hierarchy through typography and space**: Use font size, weight, and whitespace to guide the eye — not borders and boxes everywhere.
3. **Functional color**: Color encodes meaning (feasible/infeasible, compute/memory/communication). Never use color purely for decoration.
4. **Progressive disclosure**: Show the headline answer immediately (KPI cards), details on demand (expandable sections, tabs, tooltips).
5. **Consistent visual language**: Same color means the same thing across all charts and views. Same spacing rhythm throughout.

### Accessibility
- Sufficient contrast ratios for text readability (WCAG AA minimum for body text)
- Data colors distinguishable for common color vision deficiencies (red-green: use shape/pattern as secondary encoding where possible)
- Interactive elements have clear focus states
